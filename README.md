# 暗房水印 · DARKROOM.WM

> 纯浏览器端的图片加水印工具。**你的图片绝不上传服务器**,一切都在本地用 Canvas 处理。
> 零构建、零依赖、纯原生 HTML/CSS/JS —— 把文件丢进任意静态服务器(或 GitHub Pages)即可运行。

视觉语言:**摄影暗房 × 新粗野主义(Neo-Brutalism)**。深色底叠加暗红「安全灯」氛围、胶片颗粒质感,配上硬边框、硬阴影与等宽字体标签。

---

## ✦ 功能

| # | 功能 | 说明 |
|---|------|------|
| 01 | **装片 / 上传** | 点击选择 + 拖拽上传;支持多张图片,底部缩略图条切换与删除 |
| 02 | **文字水印** | 内容(支持多行)、字体、字号、颜色、不透明度、旋转、描边(颜色 + 宽度) |
| 03 | **图片水印 / Logo** | 上传任意图片作为 Logo,可调缩放与不透明度 |
| 04 | **定位** | 九宫格快速定位 **+ 在预览图上直接拖拽**自由摆放,方向键微调(Shift 加速) |
| 05 | **平铺防盗** | 满屏斜向重复水印,间距与角度可调 |
| 06 | **批量处理** | 一套参数应用到所有图片,依次冲洗(导出) |
| 07 | **设置记忆** | 通过 `localStorage` 记住上次的全部配置(含 Logo) |
| 08 | **导出** | PNG / JPG,**保持原图分辨率**,JPG 质量可调 |
| 09 | **实时预览** | 所见即所得 —— 预览即导出结果 |

## ✦ 使用

### 本地直接用
直接用浏览器打开 `index.html` 即可(无需任何服务器)。

### 用 GitHub Pages 托管
1. 把这三个文件放在仓库根目录(已就位)。
2. 打开仓库 **Settings → Pages**。
3. **Source** 选择 `Deploy from a branch`,分支选 `main`、目录选 `/ (root)`,保存。
4. 稍候即可通过 `https://<用户名>.github.io/<仓库名>/` 访问。

## ✦ 设计与技术思路

- **绝不上传**:所有图片用 `URL.createObjectURL` 在本地读取,Canvas 处理后用 `canvas.toBlob` 在本地下载;Logo 用 `FileReader` 读为 DataURL。全程不发任何网络请求。
- **所见即所得 + 原分辨率**:始终把水印绘制到一张「与原图等像素」的离屏/在屏 canvas 上,再用 CSS 把预览缩小显示。于是**预览看到的就是导出得到的**,而导出天然保持原始分辨率。
- **尺寸相对化**:字号、Logo 缩放、边距、平铺间距等都以**图片宽/短边的百分比**表达,因此同一套参数批量套到不同尺寸的图片上,观感保持一致。
- **水印「戳」抽象**:文字与 Logo 组合成一枚可复用的「戳」(`paintStamp`),单点摆放时画一次,平铺时在旋转坐标系里铺满整图。
- **JPG 白底**:JPG 不支持透明,导出前先铺白底,避免透明区域变黑。
- **零依赖**:不使用任何框架 / 打包工具 / CDN,离线可用,适合静态托管。

## ✦ 浏览器兼容

需要支持 Canvas、Pointer Events 与 `canvas.toBlob` 的现代浏览器(Chrome / Edge / Firefox / Safari 近年版本均可)。

## ✦ 隐私

100% 本地运行。图片、Logo、配置都只存在于你当前的浏览器中,刷新或导出都不会把数据发送到任何服务器。
